# Contributing

This is a set of contribution rules for a pet project listed mainly for ourselves, used exclusively by ourselves, obvious for many, set explicitly for @shbshka who has never worked in any team before.

## Branches, commits, pull requests :innocent:


> Direct push to main is not allowed (sorry)



### Process

1. create a branch

2. make changes

3. push & pr to main

4. pr approved -> squash it! (use squash merge ok?)

5. delete branch after pr



### Branch naming

```
<type>/<issue-number>-brief-description
```

> use 'noissue' when there is no issue :moyai:

#### Commits
```
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
```



#### Types used:

- **feat**: new functionality

- **fix**: bug fix

- **chore**: maintenance or dependency work

- **docs**: changes in documentation only

- **test**: tests only

- **style**: formatting only

- **refactor**: for refactoring only :moyai:



### PR resolution

1. Explain the changes

2. Get 1 (one) approval

3. Everything resolved in a conversation
